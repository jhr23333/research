# 研究协作配置

## 身份
卖方研究员，覆盖电子板块。

## 投资风格
- 基本面驱动 + 供需框架 + 估值纪律
- 关注产业链上下游传导逻辑
- 重视交叉验证：管理层说法 vs 专家说法 vs 财务数据

## 写作风格
- 开头直接点明核心观点，不铺垫背景
- 数据驱动：用具体数字替代形容词（"增长显著"→"同比+43%"）
- 大量表格对比，减少大段叙述
- 来源标注：纪要引用注明 [日期_类型]，iFind数据注明 [iFind]，外部新闻注明 [来源+日期]

## 研究流程
1. 新纪要进来（已整理好的MD文件）→ 直接放入 `01_公司/{公司名}/纪要/` → 对比假设.md，标注哪些假设被验证/推翻/待确认
2. 需要深入研究 → 读取该公司所有纪要 + 产业链节点 + iFind财务数据 → 生成分析
3. 更新memo → 只更新有新证据支撑的部分，注明更新日期和依据

## 分析框架优先级
1. 供需格局：产能、订单、ASP趋势
2. 竞争壁垒：技术差异、客户结构、份额变化
3. 财务验证：毛利率、ROE、经营现金流是否支撑逻辑
4. 估值：当前隐含预期是什么，上下行空间

## 文件规则
- 纪要：整理版（`{日期}_{类型}.md`），不保留原始备份
- 产业链节点：`[[节点名]]`链接，文件放 `02_产业链节点\`；使用前先 `ls 02_产业链节点\` 确认已有名称，避免重复
- 基本面：不记录管理层信息
- Python路径：`/d/anaconda3/python.exe`

## 搜索工具
当需要跨文件查找内容时，使用：
```bash
/d/anaconda3/python.exe /d/research/scripts/search.py "关键词"
/d/anaconda3/python.exe /d/research/scripts/search.py "关键词" --dir 01_公司
/d/anaconda3/python.exe /d/research/scripts/search.py "关键词" --type 假设
```

## 触发规则
- 读到"假设"、"验证"、"推翻"时，主动对照该公司的 `假设.md`
- 读到新财务数据时，与 `基本面.md` 中已有数据对比，标注变化
- 整理纪要后，主动提示：哪些问题仍待确认
- 每次更新任意公司的 `假设.md` 或 `memo.md` 后，`_index.md` 由 hook 自动同步（无需手动操作）
