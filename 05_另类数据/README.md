## 信号规则

> 只有**负向/异常**信号写入假设.md，正向趋势仅在看板里记录，不追加待核实问题。

| 数据源 | 触发条件（负向） | 动作 |
|--------|--------------|------|
| 韩国半导体出口前10日同比 | 连续2月同比 < -10% | 追加至鹏鼎 假设.md H2 待核实 |
| SIA全球销售额同比 | 单月同比转负，或连续2月跌幅扩大 >5pp | 追加至芯原/鹏鼎 假设.md 行业风险 |
| 台湾PCB月营收合计同比（YoY） | 连续2月同比 < -10% | 追加至鹏鼎 假设.md H3 待核实 |
| 日本设备出货额 | 连续3月环比下滑 | 追加至鹏鼎 假设.md H2 待核实（资本开支收缩信号） |
| 国内晶圆厂招标公告 | 出现 PCB/基板 相关采购 | 追加至鹏鼎 假设.md H2 待核实（潜在订单线索，正向） |

> 注：台湾PCB用**同比**（YoY）判断趋势，环比受季节性干扰大（如春节效应）

## 说明

- 数据层：`data\` 只存原始 CSV，不做分析
- 信号层：脚本运行后自动判断是否触发阈值，触发则写入公司 `假设.md`
- 看板层：`看板\` 笔记负责可视化，通过 `![[]]` 嵌入图表
- 所有数据系列在登记表里必须填写"关联假设"，孤立数据不入库
