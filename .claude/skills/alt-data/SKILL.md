# Alt-Data 另类数据工作流

数据源登记表：`D:\research\05_另类数据\README.md`
研究库根目录：`D:\research\`

---

## 触发方式

- `/alt-data` — 拉取所有已登记数据源，检查全部信号
- `/alt-data 鹏鼎` — 只拉与鹏鼎相关的数据源
- `/alt-data 芯原` — 只拉与芯原相关的数据源

---

## 执行步骤

### Step 1：读取数据源登记表

读取 `05_另类数据/README.md`，从表格 A（iFind EDB）中提取：
- 指标名称
- 关联公司
- 关联假设编号（H#）

如用户指定了公司，只处理该公司相关行。

### Step 2a：调用 iFind EDB 拉取高频宏观数据

对每条 iFind EDB 指标，用 **`get_edb_data`** 工具取最近 3 个月数据。

取数完成后整理为：

```
| 指标 | 最新值 | 上月值 | 同比 | 环比 |
```

### Step 2b：调用 iFind search_notice 拉取资本开支公告

读取 `scripts/capex_tracker.py` 中的 `CAPEX_WATCHLIST`，对每家公司调用 **`search_notice`**：
- `time_start`：90天前
- `size`：5
- `query`：`{公司名} {signal关键词}`

扫描返回片段，提取含 `HIGH_VALUE_KEYWORDS` 的段落，记录：公司、日期、关键数值（资本开支金额/产能数字）、公告标题。

### Step 3：信号检查

对照 `05_另类数据/README.md` 的信号规则表，逐条判断是否触发：

- ✅ 未触发：正常，记录数值
- ⚠️ **触发**：标记，进入 Step 4

### Step 4：信号写入假设.md

对每个触发的信号，自动在对应公司 `假设.md` 的"待核实问题"末尾追加一行：

```
- [{今日日期}] {信号描述}（{具体数值}），需核实是否影响 {关联指标} → H{#}  [来源：iFind EDB]
```

用 **Edit** 追加，不重写整个文件。

### Step 5：输出摘要

```markdown
## 另类数据更新 {今日日期}

### 数据概览
| 指标 | 最新值 | 环比 | 同比 | 信号 |
|------|------|------|------|------|
| ... | | | | ✅/⚠️ |

### 触发信号
- ⚠️ {信号描述} → 已写入 {公司}/假设.md H{#}

### 待人工跟进（外部数据源）
- CSP 资本开支：下次财报季手动录入（{预计披露时间}）
- 招标数据：手动查询 ccgp.gov.cn
- 专利数据：patent-client 尚未接入
```

---

## 注意事项

- iFind EDB 调用：用 `search_edb` 确认指标名存在后再用 `get_edb_data` 取数
- 时间范围：只取最近 3 个月，避免全量历史数据影响速度
- 写入假设.md 时只追加，不修改已有内容
- 外部数据源（CSP capex / 招标 / 专利）本 skill 不自动处理，仅在摘要中提示
