# Research Memo 工作流

研究库根目录：`D:\research\`
详细研究风格与分析框架见：`D:\research\research-config.md`

```
D:\research\
├── 01_公司\{公司名}\
│   ├── memo.md
│   ├── 基本面.md
│   ├── 假设.md            ← 核心投资假设追踪
│   ├── 纪要\

│   ├── 研报\              ← 卖方研报整理版

```

Python 路径：`/d/anaconda3/python.exe`（Git Bash 环境）

## 操作一：新建公司覆盖

**触发**：用户说"新建覆盖XX"或"建立XX覆盖"

1. 创建目录结构：`纪要\`、`研报\`、`年报\`
2. 用 **iFind stock MCP `get_stock_info`** 获取：股票代码、总市值、行业分类（字段少、不含冗余历史数据）；再用 **`get_stock_financials`** 只取最近2年的营收、毛利率、净利率、ROE、PE——调用时明确指定 `period` 和 `fields` 参数，避免返回全量历史
3. 用 **iFind news MCP `search_news`** 搜索公司名，限定最近30天、最多10条（`search_trending_news` 返回数据量大，优先用 `search_news` 控制条数）
4. 生成 `基本面.md`（财务数据直接填入，注明来源 [iFind]）
5. 生成空白 `memo.md`
6. 生成 `假设.md`（空模板，等待用户填入初始假设）

## 操作二：整理纪要

**触发**：用户发送 docx 路径，或将整理好的 MD 直接放入 inbox，或粘贴纪要文本

**读取 docx**（用户扔了 docx 时，Git Bash执行）：
```bash
/d/anaconda3/python.exe -c "
import zipfile, sys
from xml.etree import ElementTree as ET
sys.stdout.reconfigure(encoding='utf-8')
ns = '{http://schemas.openxmlformats.org/wordprocessingml/2006/main}'
with zipfile.ZipFile(r'$FILEPATH') as z:
    xml = z.read('word/document.xml')
tree = ET.fromstring(xml)
for para in tree.iter(ns+'p'):
    line = ''.join(t.text or '' for t in para.iter(ns+'t'))
    if line.strip(): print(line)
" 2>&1 | cat
```

**输出一个文件**：
- `{日期}_{类型}.md`：整理版，用下方模板（不再保留原始备份）

**整理版模板**：
```markdown
---
date: {YYYY-MM-DD}
company: "[[{公司名}]]"
type: 管理层交流/专家交流/电话会/调研
tags: [纪要, {公司名}, {类型}]
---

1.
2.
3.

## 详细内容

### {主题一}

### {主题二}

## 与上次交流的变化

## 关联产业链节点
[[]] [[]]

## 待跟进问题
-
```

**整理完成后必须执行**：
> 完成后可运行 `/cross-check` 检查本篇与历史纪要的矛盾和待跟进问题回收情况。

1. 读取该公司 `假设.md`
2. 逐条对比本次纪要内容与现有假设，输出：
   - ✅ 哪些假设获得了新支撑（引用原文）
   - ⚠️ 哪些假设出现了矛盾信息（引用原文）
   - 🔲 哪些待确认问题有了新线索
3. 询问用户是否更新 `假设.md`，如确认则用 **Edit 只修改变化的行**（状态列、最近证据列、来源列），不重写整个文件。同时在"历史更新记录"末尾 append 一行日期+变更说明
   - 来源列格式：`[日期_类型]`，类型即纪要的 type 字段（管理层交流/专家交流/电话会等），保留口径信息，便于后续交叉验证时区分证据强度
   - 向"待核实问题"新增条目时，末尾加 `→ H#` 标注影响哪条假设（多条用空格分隔，如 `→ H2 H4`）
4. **基本面数据检查**：扫描本次纪要中出现的财务数字（营收、毛利率、净利率、ROE、PE、产能、订单量等），与 `基本面.md` 中现有数据对比，如有差异则列出：
   - 哪个指标、原值 vs 纪要新值、数据口径说明（管理层指引/实际数据/预测）
   - 询问用户是否更新 `基本面.md`，确认后用 Edit 修改对应行，注明来源 `[{日期}_{类型}]`
   - 如纪要无财务数字，跳过此步
5. 列出本次纪要中识别到的所有产业链节点（即正文和 `[[]]` 链接中提到的技术、供应商、客户、工艺等节点）。**在使用 `[[]]` 链接前，先执行 `ls D:\research\02_产业链节点\` 查看已有节点文件，优先复用已有名称（避免同义词/近义词重复，如 `[[HDI]]` vs `[[高密互联板]]`）**。检查哪些节点还没有对应文件，询问用户是否要新建。用户确认后，用产业链节点模板创建，并将本次纪要中关于该节点的关键信息填入。

## 操作三：更新 memo

**触发**：用户说"更新XX的memo"

1. 读取 `纪要\` 下所有整理版文件（不含"原始"的）
2. 读取现有 `memo.md` 和 `假设.md`
3. 只更新有新证据支撑的部分，注明更新日期和来源
4. 保留"观点变化历史"

**memo 模板**：
```markdown
---
updated: {今日日期}
company: "[[{公司名}]]"
rating: 跟踪中
---

## 一句话观点

## 核心投资逻辑

## 主要风险
-

## 待核实的关键变量
1.

## 观点变化历史
- {日期}：{变化点} [来源]
```

## 注意事项
- 基本面不记录管理层信息
- 产业链链接用 `[[节点名]]`，节点文件放 `02_产业链节点\`
- 所有引用标注来源：纪要用 [日期_类型]，iFind数据用 [iFind]，外部新闻用 [来源+日期]
- iFind MCP 未连接时用 WebSearch 替代，注明来源
