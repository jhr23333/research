# Discover 覆盖池外候选发现工作流

研究库根目录：`D:\research\`
候选池输出：`D:\research\03_主题\候选池\{YYYY-WW}.md`
扫描脚本：`D:\research\scripts\discover\`（五个维度各一个脚本 + summarize.py，v1+v2 已实现）
现有覆盖池：`D:\research\_index.md`

---

## 设计原则（token 预算核心）

**全部打分与筛选在 Python 脚本里完成，LLM 不介入 400 支股票的维度计算。**

分工：
| 环节 | 执行方 | 说明 |
|------|-------|------|
| 拉取全市场数据 | Python + iFind | `scripts/discover/*.py` 独立运行 |
| 五维度打分排序 | Python | 每维度输出 Top 10 JSON |
| 合并摘要 md | Python (summarize.py) | 自动算交集 + 渲染单一 md |
| 去重与交叉判断 | LLM | 只读 `summary_YYYYMMDD.md`（约 3-5K token） |
| 写候选池 md | LLM | 每个候选 1-2 句话，5-10 个候选 |

**不做加权综合打分**。每维度独立出榜，原因：权重调参没有数据基础，且不同维度反映不同 alpha 类型（景气 vs 估值 vs 覆盖稀疏），综合会稀释信号。

**扫描节奏**：周度。日度噪音大，且数据源多为月度/季度颗粒度。

---

## 触发方式

| 命令 | 行为 |
|------|------|
| `/discover` | 全维度扫描，五张榜单 + 合并候选池 |
| `/discover {维度}` | 只跑单维度（快速复查），{维度} ∈ `催化` / `估值` / `稀疏` / `产业链` / `另类` |
| `/discover review` | 不跑脚本，只读最近一期候选池 md，做增量跟踪 |

## 五个维度

### 维度① 基本面催化（scripts/discover/scan_catalysts.py，v2）

**输入**：申万电子一级股票池（剔除已覆盖、ST）
**数据源**：iFind `iwencai` 三类查询（均为绑定时间窗口的可靠字段）
- 业绩预告净利润同比（季度绑定）
- 业绩快报净利润同比（季度绑定）
- 近 30 天机构调研次数（区间字段 `区间机构调研次数[YYYYMMDD-YYYYMMDD]`）

**打分逻辑**（叠加，最高 9 分）：
- 业绩预告：>100% +4 / >50% +3 / >30% +2 / >0% +1
- 业绩快报：>50% +3 / >30% +2 / >0% +1
- 机构调研（仅在已有催化时加成）：≥20 次 +2 / ≥10 次 +1

> 预告 + 快报双确认是 v2 核心：能区分"预测高"与"已实现高"，单一信号比 v1 更稳。机构调研为弱信号，不可单独触发候选。
>
> 重大合同/扩产公告仍未接入：iwencai 时间关键词对公告类不可靠（召回全历史），SDK `THS_ReportQuery` 字段不足。v3 计划在 LLM 侧挂 MCP `search_notice` 补位。

**输出**：`scripts/discover/out/catalysts_{YYYYMMDD}.json`

### 维度② 估值被动便宜（scripts/discover/scan_valuation.py，v2）

**数据源**：
- `THS_BasicData` 批量拉 PE_TTM / PB / 总市值 / ROE_TTM
- `THS_DateSerial` 批量拉 3 年周频 PE_TTM 序列（多代码批量，每批 50 支）

**打分逻辑**：
- PE 板块分位 < 30% 且 PB 板块分位 < 40%（横向便宜）：+3
- PE 自身 3 年历史分位 < 20%（纵向便宜）：+2
- ROE TTM > 行业中位数 且 ROE TTM > 5%（便宜但不是垃圾）：+2
- 市值 ∈ [30, 300] 亿：+1
- 剔除：PE ≤ 0 / PB 缺失 / ROE TTM < 0

> v2 增量：横向 + 纵向 + 质量三层过滤，避免落入"持续便宜的价值陷阱"。性能：先用板块分位 < 50% 粗筛（约 170 支），仅对粗筛后标的拉 3 年周频历史，全脚本 ~15s。

**注意**：单独拉出榜但**不可独立决策**——便宜的本来就便宜。必须与维度①③④交叉后才有信号价值。

### 维度③ 覆盖稀疏度（scripts/discover/scan_coverage.py）

**数据源**：iFind `iwencai` 查"近6个月研究报告数量" + `THS_BasicData` 拉市值
**打分逻辑**：
- 近 6 个月研报数 ∈ [1, 5]：+2（0 通常是垃圾股）
- 同时命中维度① 催化榜：+3
- 市值 ∈ [30, 80] 亿（被忽略区）：+1
- 研报数 > 10：剔除

**原则**：低覆盖 + 有催化 = 目标区域；低覆盖 + 无催化 = 忽略。

### 维度④ 产业链溢出（scripts/discover/scan_supply_chain.py）

**数据源**：`02_产业链节点/*.md` 正文 + iFind `iwencai` 做公司名→代码解析
**打分逻辑**：
- 被节点 md 列为供应商/客户/厂商：+3（基础分）
- 同时命中维度① 催化榜：+2
- 同时命中维度③ 稀疏榜：+1

**v1 实现**：正则提取 md 中带"股份/电子/科技/光电/半导体/新材"等后缀的中文词 + 内置白名单（常见电子公司名），再用 iwencai 解析代码。

**此维度是工作区独有的 alpha 来源**——用已覆盖公司的深度理解推未覆盖标的。节点 md 越丰富，召回越大。

### 维度⑤ 另类数据反向映射（scripts/discover/scan_alt_mapping.py）

**数据源**：`scripts/discover/alt_signals_config.json`（人工维护）+ iFind `iwencai` 查环节代表标的

**v1 实现**（依赖研究员输入）：
- 配置文件登记了每个景气信号（JSMA / SIA / 台湾 PCB / 韩国出口等）的阈值和对应行业关键词
- 研究员每月跑完 `/alt-data` 后手动更新 `latest_yoy` 字段
- 脚本读到 `latest_yoy > yoy_threshold` 的信号时，用 iwencai 查该环节电子池内标的，剔除已覆盖

**打分**：
- 信号 yoy 超阈值 + 10pp（强景气）：+3
- 信号 yoy 超阈值（一般景气）：+2
- 命中多个信号：+1

> v2 计划：自动调 `THS_EDBQuery` 拉宏观时序，脚本自己算同比，无需人工维护。**当前阻塞**：iFinD EDB 接口本月配额已用尽（错误码 -4318），自动化暂缓，仍走配置文件方案。

## 执行步骤

### Step 1：调度脚本

按命令调度：
- `/discover`：`python scripts/discover/run_all.py`，串行跑五个脚本 + 自动生成摘要 md（首次约 35s，缓存命中后约 27s）
- `/discover {维度}`：`python scripts/discover/run_all.py --only {catalysts|valuation|coverage|supply_chain|alt_mapping}`（仍会刷新 summary md，把新跑的维度与今日已存在的其他维度 JSON 合并）

> **串行执行原因**：维度③④ 需要交叉维度① 的输出（通过 out/ 下的 JSON 传递），所以必须在 ① 之后运行。`run_all.py` 的顺序是 ① → ② → ③ → ④ → ⑤ → summarize。

**LLM 在此阶段不做任何计算**，只负责触发 Bash 命令并等待摘要文件生成。

### Step 2：读取摘要 md

**只读** `scripts/discover/out/summary_{YYYYMMDD}.md` 这一个文件。

文件结构：
- 强候选表（≥2 维度命中，已自动算好交集与综合分）
- 5 个单维度 Top 10 表
- 元信息（每维度命中数、池子规模、缓存状态）

**总 token 预算**：单文件约 3-5K token（vs v1 读 5 个 JSON 约 8-12K）。

> 不要再去读 `out/*_{YYYYMMDD}.json`——summarize.py 已把所有信息合并。仅在调试单维度数据时才读 JSON。

### Step 3：交叉判断与去重

摘要 md 已经做好交集计算，LLM 只需做语义判断：
1. **审视强候选**：检查每个 ≥2 维度命中的标的是否合理（如剔除明显主题股泡沫、明显业绩翻转单点）
2. **去重已覆盖**：脚本已过滤 `_index.md`，但人工审视一遍兜底
3. **产业链节点标注**：用 Obsidian 双链 `[[节点名]]` 关联到 `02_产业链节点/` 下相关节点

### Step 4：写候选池 md

输出到 `03_主题/候选池/{YYYY-WW}.md`，模板：

```markdown
# 候选池 {YYYY年第WW周} — {扫描日期}

> 扫描维度：① 催化 ② 估值 ③ 稀疏 ④ 产业链 ⑤ 另类数据
> 本期候选数：{N}

## 强候选（≥2 维度命中）

### 1. {公司名}（{代码}）
- **命中维度**：① 催化 + ④ 产业链溢出
- **关键信号**：{1-2 句话}，例如"2026Q1 业绩预告超预期 60%，且为已覆盖 [[鹏鼎控股]] 的上游铜箔供应商"
- **关联节点**：[[AI服务器PCB]] / [[铜箔]]
- **下一步**：纳入覆盖 / 观察 / 否决
- **状态**：待决定

### 2. ...

## 单维度候选（仅供参考）

| 公司 | 命中维度 | 关键信号 |
|------|---------|---------|
| ... | ③ 稀疏 | 覆盖 3 家券商 + 近期扩产公告 |

## 本期否决（与历史否决理由一致）

- {公司名}：与 {YYYY-WW} 否决原因相同（{原因}），不再重复推送
```

### Step 5：输出终端摘要

简短列表形式，终端打印：
- 强候选数 N，单维度候选数 M
- 列出强候选前 3，每个一句话
- 提示："详见 `03_主题/候选池/{YYYY-WW}.md`，review 后在状态栏标注 纳入/观察/否决"

---

## 历史状态追踪（避免重复推送）

候选池 md 本身就是账本。每份 md 文件里的"状态"字段有四种值：
- `待决定`：新推送，未 review
- `纳入覆盖`：已走 `/research-memo` 初始化
- `观察中`：暂不深度，保留在下次扫描参考
- `否决-{原因}`：否决并写明原因

**冷却期规则（v2 待实现）**：v1 未实现，扫描每次跑出全榜。v2 计划让脚本读 `03_主题/候选池/*.md`，若某标的最近 3 期均为"否决-{同一原因}"，此维度不再推送该标的 4 周。

---

## 注意事项

- **iFind 调用预算**：五脚本合计单次扫描约 50-80 次 iFind 调用（每标的 0.2 次），周度运行可接受
- **不处理非电子板块**：申万电子一级以外不扫描（科研聚焦）
- **脚本失败容忍**：单维度脚本失败不阻塞其他维度，在摘要中标注"维度X 失败"
- **不自动生成深度**：候选池只出短名单，研究员确认后才走 `/research-memo`
- **与 /alt-data 区别**：/alt-data 跟踪已覆盖公司的持续信号，/discover 发现未覆盖公司
- **与 /compare 区别**：/compare 对比已覆盖公司之间，/discover 发现池外新标的
- **与 /kb-research 区别**：/kb-research 对已知标的做深度，/discover 产出"值得 /kb-research 的候选"

## 脚本实现（v1 已完成）

```
scripts/discover/
├── run_all.py                  # 串行调度五个维度 + 末尾自动跑 summarize（首次 36s / 缓存命中后 27s）
├── summarize.py                # 合并五维度 JSON 为单一 markdown 摘要（v2 新增）
├── scan_catalysts.py           # 维度① 业绩预告 + 业绩快报 + 机构调研（v2）
├── scan_valuation.py           # 维度② PE/PB 板块分位 + 自身 3 年历史分位 + ROE TTM（v2，带历史缓存）
├── scan_coverage.py            # 维度③ 研报覆盖数
├── scan_supply_chain.py        # 维度④ 节点 md 启发式
├── scan_alt_mapping.py         # 维度⑤ 依赖配置（EDB 配额受限）
├── common.py                   # 股票池、iFind 封装、覆盖池读取
├── alt_signals_config.json     # 另类数据信号配置（研究员维护）
├── cache/                      # 股票池缓存 + valuation 历史序列周缓存（gitignore）
│   ├── sw_electronics_pool.json        # 申万电子池，7 天过期

    ├── {dimension}_YYYYMMDD.json   # 5 个维度的原始打分明细
    └── summary_YYYYMMDD.md         # ★ LLM 消费入口
```
