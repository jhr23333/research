## 执行步骤（完整流程，`/model {公司名}` 专用）

### Step 1：读取研究材料

依次读取（**不要读所有纪要，只读最近2篇**，假设.md的"最近证据"列已是历史纪要摘要）：
1. `01_公司\{公司名}\假设.md` — 当前 H# 假设及状态（含历史结论）
2. `01_公司\{公司名}\基本面.md` — 历史财务数据
3. `01_公司\{公司名}\纪要\` — **仅按文件名日期倒序取最近2篇**，提取含具体数字的量化指引
4. `01_公司\{公司名}\研报\` — 卖方盈利预测（如有）
5. `01_公司\{公司名}\模型.md` — 已有模型（如存在，做增量更新；否则从模板新建）

从纪要中提取量化数据点，整理为驱动变量候选表：

```
| 变量 | 数值 | 口径 | 来源 |
| 2026年AI服务器营收目标 | 50-60亿 | 管理层指引 | [2026-04-01_管理层交流] |
```

### Step 2：拉取 iFind 实时数据

**两个并行批次，不超过3次调用**：

**批次1（公司本身，并行）**：
- `get_stock_performance`：当前股价、市值、PE(TTM)、PB、近1年高/低价；**同时**请求 RSI(14)、MACD柱状图（BAR）、MA20、MA60 ← 直接拉技术指标，**不要拉原始价格序列**
- `get_stock_financials`：最近2年年报，营收、**合并口径毛利率**、归母净利润、EPS、总股本

**批次2（可比公司，1次调用）**：
- `get_stock_performance`：手动指定3-4家可比公司（从 `02_产业链节点\` 或 `基本面.md` 竞对栏获取代码），**只请求**：总市值、PE(TTM)、FY1前瞻净利润（用于计算前瞻PE）
- **不要使用 `search_stocks`**——按条件搜索经常失败且耗费tokens

> 可比公司识别：优先查 `基本面.md` 的"竞对"字段，再查 `02_产业链节点\` 相关节点文件。

### Step 3：构建三情景 P&L

基于 Step 1 提取的驱动变量，按 Base/Bull/Bear 分别估算：

**情景定义**（与假设.md 状态挂钩）：
- **Bull**：所有 ✅ 假设均超预期兑现 + 🔲 待确认假设全部成立，驱动变量取乐观端
- **Base**：✅ 假设按预期兑现，🔲 假设保守打折（取管理层指引的 70-80%），驱动变量取中值
- **Bear**：⚠️ 存疑假设兑现或核心 ✅ 假设不及预期，驱动变量取悲观端

**P&L 计算顺序**：
1. 各业务线收入（量 × 价，或市场份额 × 市场规模）
2. 合并营收 = Σ 各业务线
3. 毛利润 = 营收 × 毛利率（分业务线加权）
4. OPEX = 研发费用 + 销售管理费用（参考历史费用率趋势 + 管理层指引）
5. EBIT = 毛利润 - OPEX
6. 净利润 = EBIT × (1 - 有效税率) ± 非经常性项目
7. EPS = 净利润 / 总股本

**计算辅助**：调用 `scripts/model.py --mode scenario --rev-base {基准年营收亿} --growth {增速如0.18} --gm {毛利率如0.23} --opex {费用率如0.105}`，输出三情景对比表。

### Step 4：估值与目标价

**方法选择原则**（按市场惯例）：

| 公司类型 | 主要方法 | 辅助方法 |
|---------|---------|---------|
| 成长期亏损（如半导体设计） | 前瞻 PS（1-2年后） | PEG、EV/Sales |
| 盈利成长型（如PCB制造） | 前瞻 PE | EV/EBITDA、PB |
| 成熟稳定型 | PE + 股息率 | DCF |

**目标倍数设定**：参考 Step 2 拉取的可比公司当前估值中位数，结合公司相对成长性给予溢价或折价（±20-30%）。

**目标价 = 目标市值 / 总股本**，三情景各出一个目标价区间。

### Step 5：当前定价反推（What's Priced In）

给定当前市值，反向计算市场隐含假设：

```
当前市值 / 主要估值倍数 = 市场隐含利润/营收
对应增速 = (隐含利润 - 历史利润) / 历史利润
```

对比分析：
- 隐含增速 vs. Base 情景增速 → 差距 = 市场低估/高估幅度
- 隐含毛利率 vs. 管理层指引毛利率 → 市场是否相信毛利率改善故事
- **结论句式**："市场当前 price in 的是 {情景} 假设，{尚未/已经}计入 {具体假设X}，若 {催化剂} 兑现，存在 {N%} 重估空间。"

调用 `scripts/model.py --mode implied --mktcap {当前市值亿元} --multiple {估值倍数如37} --metric {Base净利润亿元} --hist-metric {上年实际净利润亿元}`。

### Step 6：技术面信号

技术指标**已在 Step 2 批次1中一并拉取**（RSI/MACD/MA20/MA60），无需额外调用。直接使用这些数值填入信号表。

> **不要**拉原始价格序列再跑 model.py——iFind 提供现成的技术指标日频数据，直接调用比原始序列计算更准确且节省大量 tokens。`scripts/model.py --mode technical` 仅在 iFind 不可用时作为备选。

技术指标参考阈值：

| 指标 | 参数 | 信号阈值 |
|------|------|---------|
| RSI | 14日 | <30 超卖，>70 超买，30-70 中性 |
| MACD | 快线EMA12，慢线EMA26，信号线EMA9 | 金叉（MACD上穿信号线）/死叉（下穿） |
| MACD 柱状图 | Histogram = MACD - 信号线 | 正→负转向预警，负→正动能回升 |
| 移动均线 | MA20/MA60/MA250 | 价格与三条均线的位置关系 |
| 量价关系 | 5日均量 vs 20日均量；成交量与价格变化方向 | 价涨量增=健康；价涨量缩=背离预警 |

**综合判断逻辑**：
- RSI 超卖 + MACD 金叉 + 价格站上 MA20 → 技术反转信号，情绪面开始配合
- RSI 超买 + MACD 死叉 → 短期注意调整风险，即使基本面好也需等待
- 量价背离（价格创新高但成交量萎缩）→ 上涨可持续性存疑
- 技术面信号**只作辅助**，不覆盖基本面判断，只影响**时机选择**

### Step 7：写入/更新 模型.md

- **新建**：从 `99_模板\模型.md` 复制，填入所有计算结果，保存至 `01_公司\{公司名}\模型.md`
- **更新**：用 **Edit** 只修改变化的部分（假设表、P&L、技术面信号），不重写整个文件
- 在"模型更新记录"末尾追加一行：`| {今日} | {更新内容} | {触发原因} |`

## 轻量更新流程

### `假设更新` 执行步骤

1. 读取 `假设.md`，标记本次变化的 H#（状态切换或新数字）
2. 读取旧 `模型.md` 第一节（业务拆分与驱动变量表）和第二节（三情景P&L）
3. 对照变化的 H#，判断影响哪条收入线/成本假设，在哪个情景下如何调整
4. 用 `scripts/model.py --mode scenario --rev-base {基准年营收} --growth {增速} --gm {毛利率} --opex {费用率}` 验算调整后的 P&L 数字
5. 用 **Edit** 只修改 `模型.md` 中受影响的单元格，末尾更新记录追加一行

### `定价` 执行步骤

1. 调用 `get_stock_performance` 获取当前市值/股价
2. 读取旧 `模型.md` 第三节（目标价测算）取 Base 情景各指标
3. 用 `scripts/model.py --mode implied --mktcap {当前市值亿元} --multiple {Base目标PE} --metric {Base净利润亿元} --hist-metric {上年实际净利润亿元}` 反推隐含假设
4. 用 **Edit** 只更新第四节表格和定价差距结论，末尾更新记录追加一行

### `技术面` 执行步骤

1. 调用 `get_stock_performance` 请求：RSI(14)、MACD柱状图（BAR）、MA20、MA60、5日均量、20日均量（**直接请求指标，不要拉原始价格序列**）
2. 用 **Edit** 只替换第五节信号表数值和综合判断，末尾更新记录追加一行

## 注意事项

- 驱动假设必须有来源标注（纪要日期+口径），禁止凭空捏造数字
- 三情景假设要**可区分**——如果 Bear 和 Base 差距不大，说明假设不够有张力
- 可比公司估值倍数取**当前时点**数据，不用历史均值（市场环境变了）
- 技术面信号每次 `/model {公司名} 技术面` 时自动刷新，不需要每次全量更新模型
- iFind `get_stock_performance` 返回的价格数据如果不足 300 日，用实际可用天数，RSI/MACD 窗口期相应标注
