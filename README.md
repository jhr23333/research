# 研报自动化分析工作区

这是一个面向二级市场电子/半导体行业的自动化研报分析与知识库系统。

## 运行原理

本工作区采用**假设驱动（Hypothesis-Driven）**的研究框架，结合 AI 自动化处理流程，实现从原始纪要到深度研报的闭环：

1. **资料落地**：整理好的纪要（MD或docx）放入 `inbox/`，年报/研报PDF也放这里
2. **知识沉淀**：
   - **纪要归档**：整理后的纪要放入 `01_公司/公司名/纪要/`
   - **假设验证**：对比纪要内容，更新 `假设.md` 中的验证状态（✅验证中、⚠️存疑、❌推翻）
   - **Memo 更新**：将最新的逻辑变动更新至 `memo.md`
3. **全景索引**：通过 `_index.md` 实时查看所有覆盖公司的投资确定性（Conviction）和待核实问题

## 目录结构

- `01_公司/`：核心研究对象，包含基本面、核心假设、研究 Memo 和历史纪要
- `02_产业链节点/`：跨公司的技术逻辑或行业节点（如：mSAP 技术、正交背板）
- `03_主题/`：行业大的专题研究、讨论记录，以及无法归属单一公司的行业级原始资料
- `99_模板/`：各类研究文档的标准化格式定义
- `inbox/`：待处理文件的暂存区，处理完毕后清空
- `scripts/`：自动化辅助工具（全文搜索等）

## 可用工具（Claude Code skills）

- `/research-memo`：新建公司覆盖 / 整理纪要 / 更新memo
- `/kb-research`：基于知识库深度研究某公司
- `/reflect`：反方挑战投资假设
- `/kb-add`：PDF/Word/PPT 转 Markdown 并归档
- `/compare`：跨公司横向对比
- `/lint`：知识库健康检查（找孤岛节点、僵尸假设、数据矛盾）
- `/cross-check`：纪要交叉验证（历史待跟进问题回收、跨口径矛盾检测）

## 修改入口

- **新增关注公司**：说"新建覆盖XX"，Claude 会自动创建文件夹结构并拉取财务数据
- **调整研究框架**：修改 `research-config.md`
- **修改 AI 行为**：修改 `CLAUDE.md`
